docoff-react-base

This elements allows to define code that is common for all docoff-react-preview elements on the given page. This is handy for defining code fragments susch as constants or placeholder React components that can be used further on.

Usage


<textarea is="docoff-react-base">
// Your JS code here
</textarea>

Examples


<textarea is="docoff-react-base">
  const MyHello = ({name}) => (
    <strong>Hello, {name}</strong>
  );
</textarea>

docoff-react-preview

This element renders a live preview of JSX code.

Usage

We simply need to include it in the page:


<textarea is="docoff-react-preview" data-css="/shadow.css">
  <YourCustomJSX />
</textarea>

docoff-react-props

This elements renders the component prop definition table.

The props can be loaded from multiple files. This allows for writing more DRY code when several components share same props.

There are simple rules for loading props from multiple files:

  • Files get processed sequentially
  • The result of merging all previous files is used as base when merging in the current file
  • When a prop is not defined in the current file, it will not be present in the merge result
  • When a prop is defined in the current file, with docblock description of the prop type definition, the definition from current file will be used
  • When a prop default value is defined in the current file, with no docblock description of the prop type definition, the definition from base will be used, but with the default value from the current file
  • When a prop is defined in the current file, with no docblock description of the prop type definition, the definition from base will be used including default value

Examples

Relative URL with Overloading


<docoff-react-props data-src="
  /exampleJS/common.props.js
  |/exampleJS/Greeting.js
"></docoff-react-props>

Absolute URL


<docoff-react-props
  data-src="https://raw.githubusercontent.com/react-ui-org/react-ui/master/src/lib/components/SelectField/SelectField.jsx"
></docoff-react-props>
